Descripción Técnica

ARP Scan es una herramienta para descubrir hosts activos en una red local mediante el protocolo ARP (Address Resolution Protocol). A diferencia de herramientas como Nmap que envían paquetes a nivel de red, ARP Scan opera a nivel de enlace de datos, lo que lo hace muy eficiente en redes locales.

Cuándo se Usa

  • Descubrimiento inicial en redes locales
  • Cuando los escaneos ICMP están bloqueados
  • Identificación rápida de dispositivos en la red
  • Auditorías de red interna
  • Detección de dispositivos no autorizados

Ventajas

  • Muy rápido en redes locales
  • No requiere privilegios root en muchos casos
  • Funciona incluso con firewalls activos
  • Detecta hosts que no responden a ping
  • Proporciona direcciones MAC

Comandos Clave

Escaneo Básico

# Escaneo de red completa
arp-scan 192.168.1.0/24

# Escaneo con interfaz específica
arp-scan -I eth0 192.168.1.0/24

# Escaneo de rango específico
arp-scan 192.168.1.1-192.168.1.100

Opciones Avanzadas

# Mostrar información de fabricante
arp-scan --localnet --retry=5 --timeout=1000

# Usar archivo de direcciones MAC
arp-scan --file=mac-vendors.txt 192.168.1.0/24

# Modo verbose para debugging
arp-scan -v 192.168.1.0/24

# Especificar dirección MAC fuente
arp-scan --srcaddr=AA:BB:CC:DD:EE:FF 192.168.1.0/24

Alternativas

# Con arp-scan (instalación específica)
sudo apt install arp-scan
arp-scan -l

# Con arp (comando nativo)
arp -a

# Con nmap (ARP scan)
nmap -sn 192.168.1.0/24

# Con netdiscover
netdiscover -i eth0 -r 192.168.1.0/24

Ejemplos de Uso

Escaneo Completo

# Escaneo completo con información de fabricante
sudo arp-scan -I eth0 --localnet --retry=3

Información importante:

  • IP: Dirección del host
  • MAC: Dirección física del dispositivo
  • Fabricante: Identifica el tipo de dispositivo
  • Locally administered: MAC modificada (puede ser sospechosa)

Buenas Prácticas

  • Usar en combinación con otras herramientas
  • Documentar dispositivos conocidos
  • Buscar direcciones MAC sospechosas
  • Realizar escaneos periódicos para detectar cambios
  • Considerar el impacto en la red

Mitigaciones

  • Segmentación de red
  • Port Security en switches
  • Monitorización de ARP
  • Listas de control de acceso (ACL)
