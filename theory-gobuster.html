Descripción Técnica

Gobuster es una herramienta escrita en Go utilizada para realizar fuerza bruta contra servicios web. Puede descubrir directorios, archivos, subdominios y parámetros S3 buckets. Es rápida, eficiente y ampliamente utilizada en pruebas de penetración.

Cuándo se Usa

  • Fase de enumeración web
  • Descubrimiento de contenido oculto
  • Identificación de subdominios
  • Enumeración de S3 buckets
  • Búsqueda de archivos de respaldo

Módulos Principales

  • dir: Fuerza bruta de directorios y archivos
  • dns: Fuerza bruta de subdominios
  • s3: Enumeración de buckets S3
  • vhost: Fuerza bruta de virtual hosts

Comandos Clave

Enumeración de Directorios

# Escaneo básico
gobuster dir -u http://example.com -w /usr/share/wordlists/dirb/common.txt

# Con extensiones específicas
gobuster dir -u http://example.com -w wordlist.txt -x php,html,txt

# Con usuario-agente personalizado
gobuster dir -u http://example.com -w wordlist.txt -a "Mozilla/5.0"

# Con timeout personalizado
gobuster dir -u http://example.com -w wordlist.txt -t 50

Enumeración de Subdominios

# Fuerza bruta de subdominios
gobuster dns -d example.com -w /usr/share/wordlists/SecLists/Discovery/DNS/subdomains-top1million-5000.txt

# Con servidor DNS específico
gobuster dns -d example.com -w wordlist.txt -r 8.8.8.8

Opciones Avanzadas

# Mostrar todos los status codes
gobuster dir -u http://example.com -w wordlist.txt -s 200,204,301,302,307,401,403

# Excluir códigos específicos
gobuster dir -u http://example.com -w wordlist.txt --exclude-length 0

# Usar proxy
gobuster dir -u http://example.com -w wordlist.txt -p http://127.0.0.1:8080

Ejemplos de Uso

Escaneo Completo

# Escaneo completo con extensiones comunes
gobuster dir -u http://target.com \
  -w /usr/share/wordlists/dirb/common.txt \
  -x php,html,txt,js,css,json \
  -t 50 \
  -o scan_results.txt

Status codes importantes:

  • 200: Recurso encontrado
  • 301/302: Redirección (puede indicar directorio)
  • 403: Prohibido (recurso existe pero sin acceso)
  • 401: No autorizado (requiere autenticación)

Buenas Prácticas

  • Usar wordlists apropiadas para el contexto
  • Ajustar el número de threads según la red
  • Probar diferentes extensiones de archivo
  • Guardar resultados para análisis posterior
  • Combinar con otras herramientas de enumeración
